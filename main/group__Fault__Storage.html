Description

Functions that store the Cortex-M Exception Fault information into memory.

This section contains the functions that of the component CMSIS-View:Fault:Storage that store the Cortex-M Exception Fault information in memory.

◆ ARM_FaultExit()

ARM_FaultExit ( void  )

Callback function called after fault information was saved.

Callback function called after saving the fault information (see ARM_FaultSave). The default implementation will do a System Reset. If System Reset is not desired please provide a custom implementation of this function.

◆ ARM_FaultOccurred()

ARM_FaultOccurred ( void  )

Check if the fault occurred and if the fault information was saved properly.

Returns
status (1 = fault occurred and valid fault information exists, 0 = no fault information saved yet or fault information is invalid)

Call during startup to check if a fault occurred that can be sent, e.g. using ARM_FaultPrint or ARM_FaultRecord. Consider calling ARM_FaultClear after sending the fault buffer to prevent resending.
